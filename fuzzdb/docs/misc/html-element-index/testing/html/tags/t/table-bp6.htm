Assertion:
I finally discovered the answer to a strange behavior that I had seen for a long time in Netscape: occasionally, some text was unselectable. The answer: the content of tables in Netscape (text especially), is not selectable by the reader if the table is floated (using the ALIGN="left" or ALIGN="right".) This behavior has been the case ALL the way back to Netscape 2.0.

Plain table, no align attribute:
filler text
TD Cell 1TH Cell 2
TH Cell 3TD Cell 4
filler text


Plain table, align=left attribute:
filler text
TD Cell 1TH Cell 2
TH Cell 3TD Cell 4
filler text


Plain table, align=right attribute:
filler text
TD Cell 1TH Cell 2
TH Cell 3TD Cell 4
filler text
